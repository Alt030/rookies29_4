## 🎯 Repository Guide

* **브랜치 전략**
  * `main` ← 배포용
  * `develop` ← 통합 개발
  * `server` / `client` ← 기능별 브랜치
* **커밋 컨벤션 (Conventional Commits)**
  * `feat`: 새로운 기능 추가
  * `fix`: 버그 수정
  * `refactor`: 리팩토링
  * `docs`: 문서 작성
  * `chore`: 설정 / 기타 작업
* **PR 규칙**
  * 작업 단위별 PR → 최소 1명 리뷰 후 머지
  * 템플릿 기반 (PR 설명, 체크리스트, 관련 이슈 링크 포함 필수)
